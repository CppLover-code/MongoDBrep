MongoDB - одна из самых популярных NoSQL баз данных.

1. Сравнение: 
PostgreSQL — это SQL база.

Там всё хранится в таблицах:

id	name	age
1	Mark	25

Строгая структура:

есть колонки
типы данных
связи между таблицами

NoSQL = Not Only SQL

В MongoDB данные хранятся не в таблицах, а в документах.

Пример документа:

JSON
{
    "name": "Mark",
    "age": 25,
    "skills": ["Python", "Flask"],
    "is_admin": true
}
Это очень похоже на Python dict.

2. Главная идея MongoDB

MongoDB:

гибкая
быстрая
удобна для больших объемов данных
не требует строгой схемы

Можно хранить разные структуры в одной коллекции.

Например:

{
    "name": "Alice"
}

и

{
    "name": "Bob",
    "age": 30,
    "city": "London"
}

В SQL так нельзя без NULL колонок.

3. Термины MongoDB
SQL	            MongoDB

Database	    Database

4. Когда используют MongoDB

MongoDB хороша когда:

Гибкая структура данных

Например:

соцсети
чаты
блоги
игры
аналитика
Очень много данных

MongoDB хорошо масштабируется.

Часто меняется структура проекта

Например стартап:

сегодня у пользователя 3 поля
завтра 20 полей

5. Когда лучше SQL

SQL лучше когда:

важны связи между таблицами
сложные JOIN запросы
банковские системы
строгая структура данных
транзакции
